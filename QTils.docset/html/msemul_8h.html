int GetThreadPriority ( HANDLE  hThread)

get a thread's priority value. The current POSIX sched_priority is mapped onto the range of Microsoft values THREAD_PRIORITY_{LOWEST,BELOW_NORMAL,NORMAL,ABOVE_NORMAL,HIGHEST} . A check is then made to see if that value maps back onto the current POSIX sched_priority; if not, the POSIX priority is adjusted so that an expression like
SetThreadPriority( B, GetThreadPriority(A) );
always gives 2 threads with identical priorities.

DWORD SuspendThread ( HANDLE  hThread)

Suspend the given thread. On Apple/MACH, the suspend/resume feature of the underlying Mach threads is used. On other systems, the thread is sent an interrupt that should suspend it while it tries to obtain the lock on an already locked mutex. NB: with that approach it is thus not possible for a thread to suspend itself!

Wait for an event to occur on hHandle, for a maximum of dwMilliseconds
semaphore: wait to lock the semaphore

Returns WAIT_OBJECT_0 on success, WAIT_TIMEOUT on a timeout and WAIT_ABANDONED on error.
